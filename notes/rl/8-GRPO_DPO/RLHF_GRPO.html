强化学习传统意义上，多应用于机器人，运筹优化领域。但随着 llm 的爆火，强化学习也随之应用到其中。将强化学习应用到 llm 文本领域微调的框架称为 RLHF。

对比维度RLHF（Reinforcement Learning with Human Feedback）传统强化学习微调（RL Fine-tuning）
奖励来源来自人类偏好，先收集人类对模型输出的排序数据，再训练奖励模型（RM）预测分数来自环境规则，通过环境直接计算奖励（例如游戏得分、任务成功率）
数据形式（Prompt，多条候选回答 + 人类排名）→ 奖励模型 → 奖励信号（状态 State，动作 Action，奖励 Reward）由环境交互直接生成
优化目标最大化符合人类偏好的生成结果，并加入 KL 惩罚，防止策略过度偏离初始模型最大化环境奖励（Expected Return），不需要额外 KL 惩罚
训练流程① 监督微调（SFT） ② 奖励模型训练（RM） ③ RL（PPO等）优化策略直接使用 RL 算法（PPO、SAC 等）与环境交互优化策略
适用场景规则模糊、奖励不可编程的任务（对话、文本生成、道德价值对齐等）规则明确、奖励可编程的任务（机器人、游戏、路径规划等）
优点能对齐人类价值观，适合复杂开放任务训练目标清晰，收敛过程易解释
缺点奖励模型可能引入偏差，人工标注成本高依赖环境设计奖励函数，复杂任务难手动定义
传统 RL 微调是在“环境自带奖励”的情况下直接优化策略，

RLHF 是在“环境不给奖励”的情况下，用人类偏好训练一个奖励模型，再用 RL 优化策略。

当前大模型微调的主流技术有 GRPO（改进自 PPO），在本章节中也会重点介绍。

# NOTE(sgm): this implementation only consider outcome supervision, where the reward is a scalar.
@register_adv_est(AdvantageEstimator.GRPO)  # or simply: @register_adv_est("grpo")
def compute_grpo_outcome_advantage(
    token_level_rewards: torch.Tensor,
    response_mask: torch.Tensor,
    index: np.ndarray,
    epsilon: float = 1e-6,
    norm_adv_by_std_in_grpo: bool = True,
    config: Optional[AlgoConfig] = None,
) -> tuple[torch.Tensor, torch.Tensor]:
    """
    Compute advantage for GRPO, operating only on Outcome reward
    (with only one scalar reward for each response).

    Args:
        token_level_rewards: `(torch.Tensor)`
            shape is (bs, response_length)
        response_mask: `(torch.Tensor)`
            shape is (bs, response_length)
        index: `(np.ndarray)`
            index array for grouping
        epsilon: `(float)`
            small value to avoid division by zero
        norm_adv_by_std_in_grpo: `(bool)`
            whether to scale the GRPO advantage
        config: `(Optional[AlgoConfig])`
            algorithm configuration object

    Note:
        If norm_adv_by_std_in_grpo is True, the advantage is scaled by the std, as in the original GRPO.
        If False, the advantage is not scaled, as in Dr.GRPO (https://arxiv.org/abs/2503.20783).

    Returns:
        advantages: `(torch.Tensor)`
            shape is (bs, response_length)
        Returns: `(torch.Tensor)`
            shape is (bs, response_length)
    """
    scores = token_level_rewards.sum(dim=-1)

    id2score = defaultdict(list)
    id2mean = {}
    id2std = {}

    with torch.no_grad():
        bsz = scores.shape[0]
        for i in range(bsz):
            id2score[index[i]].append(scores[i])
        for idx in id2score:
            if len(id2score[idx]) == 1:
                id2mean[idx] = torch.tensor(0.0)
                id2std[idx] = torch.tensor(1.0)
            elif len(id2score[idx]) > 1:
                id2mean[idx] = torch.mean(torch.tensor(id2score[idx]))
                id2std[idx] = torch.std(torch.tensor([id2score[idx]]))
            else:
                raise ValueError(f"no score in prompt index: {idx}")
        for i in range(bsz):
            if norm_adv_by_std_in_grpo:
                scores[i] = (scores[i] - id2mean[index[i]]) / (id2std[index[i]] + epsilon)
            else:
                scores[i] = scores[i] - id2mean[index[i]]
        scores = scores.unsqueeze(-1) * response_mask

    return scores, scores

🧩 DPO

无论是 PPO 还是 GRPO，它们都离不开一个准确的奖赏函数 Reward Model。尤其在文本生成、情感分析等方面，都需要一个 RM 来对 rollout 文本进行排序（不然就是人为标注）。整个过程非常麻烦， DPO 放弃了 RM 的构建，直接使用偏好数据进行优化。

从第一性原理来看，模型优化的重点在于拟合真实分布，比如对于问题 xxx﻿ 的两种回答 y1,y2y_1, y_2y1​,y2​﻿ ，模型的目的就是拟合 pθ(y1,y2,x)≈p(y1≻y2∣x)p_\theta(y_1,y_2, x) \approx p(y_1 \succ y_2|x)pθ​(y1​,y2​,x)≈p(y1​≻y2​∣x)﻿（公式代表回答 y1y_1y1​﻿ 比 y2y_2y2​﻿ 更适合的分布）。直接处理该问题，难点在于模型该如何表示 pθ(y1,y2,x)p_\theta(y_1, y_2,x)pθ​(y1​,y2​,x)﻿

在 VAE 中，模型表示分布的方式为：先将分布建模为高斯分布 N(μθ(x);Σθ(x))\mathcal N\left(\mu_\theta(x) ; \Sigma_\theta(x) \right)N(μθ​(x);Σθ​(x))﻿ ，模型接收输入 xxx﻿ ，最终仅输出 μθ(x),Σθ(x)\mu_\theta(x), \Sigma_\theta(x)μθ​(x),Σθ​(x)﻿

DPO 中，它引入一个先验知识，即 r(x,y)r(x,y)r(x,y)﻿ 就是强化学习中的 RM

在 Bradley-Terry model 的理论中， p(y1≻y2∣x)p(y_1\succ y_2|x)p(y1​≻y2​∣x)﻿ 是真实存在的，通过其可以反推出最优的 r(x,y)r(x,y)r(x,y)﻿。同时 r(x,y)r(x,y)r(x,y)﻿ 还和最优策略 π(y∣x)\pi(y|x)π(y∣x)﻿ 满足
